Interactivity#

Interactive messages#

Pumble's interactive messages enhance communication by allowing users to perform actions directly within the message interface. These messages can include buttons, select menus, and other interactive elements, enabling users to respond to requests, provide feedback, or trigger workflows without leaving the conversation.

Compose your interactive message#

Adding interactive components is similar to adding any other block. The following example provides insight on how to add buttons in an actions block within a message payload:

1await ctx.say({
2    text:"John requests read permission for the header design document.",
3    blocks:[
4        {
5            type:"rich_text",
6            elements:[
7                {
8                    type:"rich_text_section",
9                    elements:[
10                        {
11                            type:"text",
12                            text:"John requests read permission for the header design document."
13                        }
14                    ]
15                }
16            ]
17        },
18        {
19            type:"actions",
20            elements:[
21                {
22                    type:"button",
23                    onAction:"approve_btn",
24                    value: "some-metadata",
25                    text:{
26                        text:"Approve",
27                        type:"plain_text"
28                    },
29                    style:"primary"
30                },
31                {
32                    type:"button",
33                    onAction:"reject_btn",
34                    text:{
35                        text:"Reject",
36                        type:"plain_text"
37                    },
38                    style:"danger"
39                }
40            ]
41        }
42    ]
43}, "in_channel");

You can replace buttons with other interactive elements based on your needs. After composing and sending such a message, it should soon be visible in your channel.

Handle interactive payload#

When a user interacts with an interactive component published by your app in Pumble, block interaction payload will be sent to your app. This payload notifies your app about the interaction and provides a bundle of information about relevant details of the interaction. Understanding this payload is crucial for your app to process interactions effectively.

Pumble delivers this payload to your app's configured handlers. The payload includes all the contextual information your app requires to determine the type and specifics of the interaction.
Inside the payload, there is a value field that represents the actionable input value. For example, it corresponds to the value field from a button or the value field from a selected option if a select menu is used.
Interaction payload for previous message when Approve button is pressed:

1interactionPayload = {
2    workspaceId: "668478600000000000000000",
3    userId: "668478600000000000000000",
4    channelId: "668478600000000000000000",
5    sourceType: "MESSAGE",
6    sourceId: "668478600000000000000000",
7    actionType: "button",
8    onAction: "approve_btn",
9    payload: { value: "some-metadata" } as string
10}

Take a look at following example on how to handle interactive payload based on previous message.

Inside your app's definition (like here), add following code:

1blockInteraction: {
2    interactions: [
3        {
4            sourceType: "MESSAGE",
5            handlers: {
6                "approve_btn": async ctx => {
7                    await ctx.ack();
8                    console.log(ctx.payload);
9                    // additional stuff you want to do
10                    await ctx.say({text: "Approved"}, "in_channel", true);
11                },
12                "reject_btn": async ctx => {
13                    await ctx.ack();
14                    console.log(ctx.payload);
15                    // additional stuff you want to do
16                    await ctx.say({text: "Rejected"}, "in_channel", true);
17                }
18            }
19        }
20    ]
21}
NOTE

approve_btn handler must correspond to the button's onAction field to correctly handle button interaction events.

Dynamic select menus#

Dynamic select menu is interactive element that loads its options from an external data source, allowing for a dynamic list of options. Each time a select menu of this type is opened or the user starts typing in the typeahead field, we'll send a request to your app. For more details about dynamic select menu properties, see this.

Your app should provide handlers that will be called each time user interacts with dynamic select menu. The handler should return an array of options or option groups. See the following example.

Inside your app's definition (like here), add following code:

1dynamicMenus: [
2    {
3        onAction: "dynamic_menu_1",
4        producer: async (ctx) => {
5            return [
6                {text: {type: "plain_text", text: "Option 1"}, value: "1"},
7                {text: {type: "plain_text", text: "Option 2"}, value: "2"}
8            ]
9        }
10    }
11]
NOTE

onAction must match the onAction field of the dynamic select menu, otherwise the handler won't be called.

The provided handler will be executed when the following message is sent:

1await ctx.say({
2    text:"Dynamic select menu",
3    blocks:[
4        {
5            type:"actions",
6            elements:[
7                {
8                    type: "dynamic_select_menu",
9                    min_query_length: 3,
10                    onAction: "dynamic_menu_1",
11                    placeholder: {text: "Select something", type: "plain_text"}
12                }
13            ]
14        }
15    ]
16}, "in_channel");
